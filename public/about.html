QuickDCP in one paragraph

QuickDCP is a focused pipeline for turning mezzanine picture, audio, and subtitles into cinema‑ready DCPs with a cryptographically verifiable proof chain. This node is the local control surface: you upload assets, launch jobs, monitor workers, and publish proofs that festivals and cinemas can verify independently.

FFmpeg / OpenDCP based encoding
S3 ingest + out buckets
Time‑stamp authority (TSA) proofs
FilmPassport integration
Zero‑trust storage model

Node architecture

How this node fits between your editorial tools and the cinema server.

  • Ingest. Files land in an S3 ingest bucket via multipart upload.
  • Jobs. The API records render jobs, profiles, and customers in Postgres.
  • Worker. A stateless worker pulls jobs, runs ffmpeg / OpenDCP, and writes DCP packages.
  • Manifests. Each job emits a signed manifest JSON with hashes for all reels and assets.
  • Proofs. Manifests are locked by TSA + FilmPassport so anyone can prove they were not tampered with.

The /public/index.html console is just one client of the API. SDKs and CI can call the same endpoints.

Designed for operators

The System Core layout mirrors how facilities actually work: separate panels for ingest, queue, QC, proof, and KDM. Everything you see on the dashboard is backed by a simple HTTP API so it can be automated or embedded into your own UIs.

  • Fast manual checks when you are on a projector or laptop.
  • Scriptable via Python / Node SDKs.
  • Safe defaults: encrypted buckets, object lock, minimal secrets.

Security & proof chain

QuickDCP assumes that storage, operators, and even this UI may eventually be compromised. Instead of trusting any single component, it builds a chain of cryptographic anchors that can be verified from the outside.

  • TSA proof binds manifest hash to a public time‑stamp.
  • FilmPassport (or similar) records provide an external proof of possession.
  • S3 Object Lock ensures manifests and proofs cannot be rewritten silently.
  • Verify API is read‑only and can be exposed publicly.

What the Verify page does

verify.html is the public entry point for festivals, distributors, and cinemas:

  • Accepts a job_id or manifest SHA‑256.
  • Queries /verify/<ref> and shows status.
  • Generates a curl snippet and QR deep link.
  • Makes it easy to compare hashes with content on the projector.

You can host verify.html on a separate origin or CDN and point it at a hardened Verify API.

FAQ

Is this a full mastering system?
No. QuickDCP is the pipeline and proof layer. It expects graded, conformed picture and audio from Resolve, Premiere, or your internal tools.

Can I run multiple workers?
Yes. Workers are stateless. Point several workers at the same queue database and S3 buckets to scale out.

Where do KDMs fit?
The KDM Matrix in the System Core is a preview surface. Actual KDM generation uses your keys and certificate authority; this UI is designed to integrate with that, not replace it.

Can festivals verify without talking to me?
Yes. If they have the manifest hash or job id and access to the Verify API, they can prove whether a DCP on disk matches what you generated.
